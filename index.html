Data Types

UFO

The data type of the “UFO” dataset is a table as it is made up of rows and columns. It is independent of any particular visualization. In this case, each row represents a specific item or information concerning a UFO sighting seen over the last century. Each column represents a specific attribute for all UFO sightings. Lastly, each cell portrays the intersection of a row and a column ultimately revealing the value of their combination, for example, the state or country, of a specific UFO sighting seen.
  • Date: Attribute
    • Attribute Type: Ordinal, Sequential
  • Time: Attribute
    • Attribute Type: Ordinal, Sequential
  • City: Attribute
    • Attribute Type: Categorical
  • State: Attribute
    • Attribute Type: Categorical
  • Country: Attribute
    • Attribute Type: Categorical
  • Shape: Attribute
    • Attribute Type: Categorical
  • Duration: Attribute
    • Attribute Type: Quantitative

Airports

The row data type of the “Airports” dataset is a network as it is made up of rows and columns and is independent of any particular visualization. In this case, each row represents a specific item or information concerning the point data of the location of a certain Airport facility in Massachusetts. Each column represents a specific attribute for the data point location of an airport facility. Each cell portrays the intersection of a row and a column ultimately revealing the value of their combination, for example, the specific latitude of the airport facility data point.
  • Airport-Name: Attribute
    • Attribute Type: Categorical
  • Latitude: Position
  • Longitude: Position

Google-Web

The row data type of the “Google Web” dataset is a network as it specifies a particular relationship between two nodes. This dataset displays the parent and receiving nodes within the google webpage which can be considered a tree since it does not have a repeating cycle. For example, you can strictly view that the FromNodeID 0 has a relationship with the ToNodeID 891835.
  • DFromNodeID: Link
  • ToNodeID: Link

Task Abstraction

High-level task: Consume, Discover

  • Used to discover target (Temperature) relative to the area (Vancouver) throughout different months throughout the year.

Medium-level task: Search, Lookup

  • We know the target (Temperature) and the location (Vancouver).

Low-level task: Query, Identify

  • Can identify a specific temperature in accordance to a specific month.

Target: Outlier

  • The data points presented within this visualization are outliers.

High-level task: Consume, Present

  • Communicates something specific to an audience.

Medium-level task: Search, Browse

  • The exact target is unkown but locations are specified.

Low-level task: Query, Compare

  • Summarizes when the temperature and displays a comparable visualization of two locations when they were above a specific degree during the course of 20 years.

Target: Similarity

  • Quantitative measurement of two attributes, two different locations are presented. It shows a similarity target as it can see how often the two locations have been above 20 degrees during the same year.
